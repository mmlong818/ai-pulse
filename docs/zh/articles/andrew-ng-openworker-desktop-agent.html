吴恩达开源 OpenWorker：只交付成果、不闲聊的本地桌面 Agent

吴恩达联手 Rohit Prasad 发布 MIT 协议的本地优先桌面 Agent，跨文件和 25+ 应用把任务做完再交付，数据不出本机，剑指大厂闭源"AI 同事"。

吴恩达发布了 OpenWorker——与 Rohit Prasad 共同打造、MIT 协议开源的桌面 Agent，在拥挤的 Agent 赛道上站位鲜明：开源、本地优先、以交付成果论英雄而不是陪聊。给它一个目标——一份成稿文档、一个清理完的收件箱、一次日历更新——它会把目标拆解成步骤，跨本地文件和 25 个以上集成（Slack、邮件、日历、MCP 服务器等）执行，在任何有后果的操作前先请示，最后交回完成品。

架构上的本地优先

OpenWorker 跑在用户自己的机器上——macOS 已上线，Windows 版在路上——Tauri 桌面壳搭配本地 Python Agent 服务，没有托管后端：用户自带 API key，数据只经由用户选定的模型服务商或集成离开本机。模型路由与供应商解耦，精选约 30 个模型：GPT-5.6、Claude、Gemini、DeepSeek、Mistral、Grok，经 Together/Fireworks 接入 Kimi、GLM 等开放权重模型，也可通过 Ollama 完全本地推理。项目基于吴恩达此前的多供应商库 aisuite，开放测试标签下上线数小时即近千星。

对着谁来的

"交付完成的工作，而不是聊天"这句定位，明显是冲着大厂正在推的闭源桌面"AI 同事"——Anthropic 的 Claude Cowork、OpenAI 的 ChatGPT Work 等。那些产品把 Agent 框架、模型和数据通路打包进一个封闭订阅；OpenWorker 把三者全部拆开：框架是开源代码，模型随便换，数据留在本地。

为什么重要：Agent 框架层正在成为今年的关键竞争层，大厂押注它将是专有的。一个来自业内最受信任的教育者、可信的开放且模型无关的替代品，恰好在这一层施加商品化压力——也给有隐私合规约束的企业一条"文件永不出本机"的桌面 Agent 路径。社区项目能否做出商业产品的完成度，现在成了一个带着公开仓库的实证问题。
