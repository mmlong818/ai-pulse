Cursor发布Router智能路由：按请求选模型，成本降三到五成

Cursor推出请求级模型路由Router，为每个编码任务自动匹配最合适的模型，早期企业客户实测成本降低30%-50%且质量不打折。

Cursor于7月22日发布Cursor Router——一个智能路由层，对每条编码请求进行分类，再分发给最合适的模型处理。官方称，该系统在线上A/B测试中以最高低60%的成本交付前沿级质量；参与早期测试的企业客户实测，相比把所有请求都发给Claude Opus 4.8，成本降低30%-50%，输出质量没有可测量的下滑。

工作原理

Router是一个请求级分类器，训练数据来自60多万条真实线上请求。它会分析查询内容、上下文、任务复杂度和所属领域，然后套用一套公开的简单逻辑：常规工作交给高性价比模型，UI改动交给设计感更强的模型，真正的硬骨头依然交给前沿推理模型。Cursor强调，省钱靠的是把简单活从贵模型手里拿走，而不是给难题降级。这个分类器还做了缓存感知——训练数据和公布的节省数字都计入了切换模型造成的缓存失效开销，而这笔账是很多路由方案悄悄回避的。

用户可在三档模式中选择：Intelligence（智能优先）、Balance（均衡）、Cost（成本优先）。按官方数据，Balance模式下每次commit成本为4.63美元，Intelligence模式为6.76美元，而全量走Opus 4.8是7.34美元。Router即日起在Teams和企业版全平台可用，管理员可按团队开启、设置模型黑白名单和默认配置。

为什么重要

模型路由这个概念创业公司讲了很久，但Cursor把它做成面向团队默认开启的功能，标志着这项技术从研究演示走向了采购清单。对于被AI编码账单压得喘不过气的企业来说，"质量不变、成本砍三到五成"把选型问题从"用哪个模型"改写成了"用哪个路由"。这对前沿实验室同样是压力：如果一个分类器就能把大多数请求分流给便宜模型，那么旗舰定价就只剩最难的那一小块任务撑着了。
