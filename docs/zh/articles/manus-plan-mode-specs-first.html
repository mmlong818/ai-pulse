Manus 上线 Plan Mode：先出方案再动手，粗糙想法变清晰规格

Manus 新增 Plan Mode，Agent 先把模糊需求扩写成可审阅的执行方案，用户确认后再开工，把编码 Agent 验证过的模式带到了通用任务。

Manus 于 7 月 22 日发布 Plan Mode：通用 Agent 在动手执行之前，先起草一份结构化方案。用户不再需要一上来就写出精心设计的提示词——系统会把粗糙的想法扩写成清晰的规格说明，供用户审阅、修改方向，确认后才开始干活。官方把这次发布概括成一句话：拿到好结果，不该以写出完美 prompt 为前提。

先规划，后执行

Plan Mode 实际上在"意图"与"自主执行"之间插入了一道检查点：Agent 先说明它理解的任务是什么、打算怎么做、交付物长什么样；用户确认或纠偏后，再按商定的规格执行。开发者对这套模式不会陌生——Claude Code 等编码 Agent 早已把"先陈述方案、再改动文件"的显式规划模式带火——而 Manus 把它推向了更宽的任务面：研究报告、网站、幻灯片、数据处理等。

这也是 Manus 7 月密集发版的延续：7 月 9 日的 Branch（同一上下文并行探索多个方向）、7 月 13 日的 Auto-Publish、7 月 14 日的原生 PPT 生成之后，又一项落地。

为什么重要：自主 Agent 最大的隐性成本，是长时间朝错误方向狂奔——浪费时间、烧掉额度、消磨信任。强制的方案审阅环节直接命中这一失败模式；它从编码工具扩散到面向大众的 Agent 产品，说明"规格先行"正在固化为 Agent 交互的标准范式。在底层模型能力趋同的当下，Agent 产品的竞争越来越集中在这一层：工作流控制、检查点设计、以及"好不好驾驭"的体验本身。
