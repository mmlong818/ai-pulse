Dorsey旗下Block推出Buzz：人与AI智能体同频道协作的开源群聊

Block开源发布基于Nostr协议的群聊平台Buzz，给每个AI智能体独立加密身份和权限，让它们以正式成员身份进驻团队频道，矛头直指Slack。

Jack Dorsey旗下的Block于7月21日发布Buzz——一款免费开源的团队群聊平台，从底层就为"人类+AI智能体"混编团队设计。Dorsey将其定位为帮助企业摆脱对Slack和GitHub的依赖。

智能体是正式成员，不是机器人

Buzz最核心的设计选择是：AI智能体是频道的正式参与者，而非靠命令唤起的bot。每个成员——无论人还是智能体——都拥有一个带独立权限范围的加密身份；平台运行在去中心化的Nostr协议上，而非中心服务器。Dorsey用四个词概括这套架构："模型无关、去中心化、自主权、开源"。除聊天外，Buzz还内置了项目管理，团队可以在智能体汇报进度的同一个窗口里跑GitHub式的工作流。

桌面应用已按Apache 2.0许可免费上线macOS、Windows和Linux。Block也坦承产品尚处早期：移动端、智能体操作的审批门禁、更完整的语音功能都还在开发中。

产品背后的一贯路线

Buzz延续了Dorsey十年来用去中心化协议对抗平台巨头的执念——此前已有Bluesky、Damus和Bitchat。但它也踩中了一个真正悬而未决的竞争窗口：当创业公司开始批量运行编程和运营智能体，人类监督它们的协作层还没有赢家。Slack为人类而生、智能体是后来"焊"上去的；Buzz把这个前提整个翻转了过来。

为什么重要

这款产品真正值得押注的不是聊天界面，而是身份模型。给每个智能体一个带细粒度权限的加密身份，直击一个正快速浮出水面的企业基础设施难题——哪个智能体做了什么、经谁授权。恰逢前沿实验室正为模型越出沙箱的行为焦头烂额的一周，一个可审计、权限收敛的协作层来得正是时候。至于一款基于Nostr的早期产品能否把团队从根深蒂固的Slack上撬走是另一回事——但如果"智能体原生工作空间"注定成为一个品类，Buzz已经先给它立好了规矩。
